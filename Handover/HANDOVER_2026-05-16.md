## Known Issues / Gotchas for Next Session

### .env UTF-8 BOM
The `.env` file was created by PowerShell with a UTF-8 BOM. Fixed by using `load_dotenv(..., encoding="utf-8-sig")` in `main.py`. Do not change this.

### dotenv load order
`load_dotenv` is called at the very top of `main.py` before any `src.*` imports. This is intentional — the module-level `_API_KEY = os.getenv(...)` in `trend_fetcher.py` must run after dotenv has populated the environment.

### CI workflow has been unreliable
The `.github/workflows/ci.yml` has failed multiple times with "Invalid workflow file" YAML errors. The current version is simplified — `on:` uses explicit keys, no colons inside run values. A `.gitattributes` file now enforces LF line endings on all YAML. If CI fails again, check the raw file in the GitHub UI for line ending or encoding issues.
